# Stage 3 — Реализация бота (Dating Bot)

## Что сделано

На третьем этапе реализован полноценный Telegram-бот для знакомств на базе **aiogram 3**, **PostgreSQL**, **Redis** и **Docker**.

## Модели БД (`bot/db/models.py`)

| Таблица | Назначение |
|---|---|
| `users` | Telegram-пользователь; хранит `telegram_id` и `username` |
| `user_profiles` | Анкета: имя, возраст, пол, город, bio, диапазон возраста партнёра |
| `user_photos` | Фото пользователя, до 5 штук, с порядком `sort_order` |
| `likes` | Лайки: `from_user_id → to_user_id`, уникальная пара |
| `matches` | Взаимные лайки; `user_a_id < user_b_id` (канонический порядок) |
| `ratings` | Три скора (L1/L2/L3) на пользователя |
| `rating_events` | Журнал событий: `like_received`, `skip_received`, `skipped` |
| `referrals` | Реферальные пары: кто кого пригласил |

## Хэндлеры

### `start.py` — команда `/start`
- Если пользователь новый → запускает FSM регистрации, сохраняет `username` и реферальный ID в state.
- Если уже есть анкета → приветствует и показывает главное меню.
- При каждом `/start` обновляет `username` в БД, если он изменился в Telegram.

---

### `registration.py` — заполнение анкеты (FSM)

Состояния: `name → age → gender → seeking_gender → city → bio`

- **`/cancel`** зарегистрирован первым (через `StateFilter`), поэтому перехватывает команду раньше шагов ввода.
- После завершения создаёт/обновляет `UserProfile`. При редактировании — старый профиль удаляется, **лайки/мэтчи/фото сохраняются**.
- Если пользователь пришёл по реферальной ссылке и регистрируется впервые → создаётся запись `Referral` и пересчитывается рейтинг пригласившего.

---

### `profile.py` — своя анкета

- **"👤 Моя анкета"** — показывает анкету с фото (одно фото с подписью, или медиагруппа для нескольких), выводит рейтинг (L1 / L2 / L3).
- **"✏️ Редактировать анкету"** — запускает тот же FSM регистрации заново; отмена через `/cancel` оставляет старую анкету нетронутой.

---

### `browse.py` — листалка анкет

Полный сценарий свайпа:

1. **"🔍 Смотреть анкеты"** → берёт следующего кандидата из Redis-очереди (или перезаполняет её из БД через `get_ranked_candidates`).
2. Показывает карточку: фото + имя/возраст/город/bio. Если фото несколько — кнопка **"📸 Все фото (N)"**.
3. **❤️ Лайк** → меняет кнопки на «Написать сообщение / Просто лайк».
4. **✍️ Написать сообщение** → FSM переходит в `BrowseStates.typing_message`, пользователь пишет текст (до 500 символов).
5. **👍 Просто лайк** → лайк без сообщения.
6. Цель получает уведомление с карточкой + кнопками **❤️ Взаимный лайк / 👎 Пропустить**. Если было сообщение — оно вставляется в уведомление.
7. Если оба лайкнули друг друга → создаётся `Match`, обоим приходит сообщение с `@username` другого.
8. **👎 Пропустить** → записывает `RatingEvent(skip / skip_received)`, показывает следующую анкету.

---

### `photos.py` — управление фотографиями

- **"📷 Мои фото"** → показывает все текущие фото и клавиатуру с кнопками.
- **Добавить фото** — загрузка через Telegram, сохраняется `file_id` как `photo_url`.
- **🗑 Удалить** — удаляет конкретное фото по номеру, перенумеровывает оставшиеся.
- **⬆️ / ⬇️ Переставить** — меняет `sort_order` двух соседних фото. Используется трёхшаговый swap через временное значение 100, чтобы не нарушить уникальный индекс `(user_id, sort_order)`.
- После каждого действия старые сообщения с фото удаляются и отправляются заново (автообновление вида).

---

### `matches.py` — мэтчи и рефералы

- **"💌 Мои мэтчи"** → список кнопок с именами мэтчей.
- Нажатие на мэтча → полная анкета (все фото + `@username` для связи).
- **"🔗 Пригласить друга"** → генерирует реферальную ссылку вида `https://t.me/<bot>?start=ref_<telegram_id>`.

## Сервисы

### `services/rating.py` — трёхуровневый рейтинг

| Уровень | Что считает | Макс. балл |
|---|---|---|
| **L1** (анкета) | Имя +1, возраст +1, пол +1, кого ищу +1, город +1, bio +1.5, диапазон возраста +1, фото ×0.5 (макс. 2.5) | 10 |
| **L2** (поведение) | Лайки полученные ×0.3, like-ratio ×3.0, мэтчи ×0.4 | 10 |
| **L3** (итог) | L1×0.4 + L2×0.6 + реф. бонус (0.5 за каждого, макс. 2.0) | ~12 |

`update_user_rating(user_id, session)` пересчитывает все три уровня и сохраняет в таблицу `ratings`. Вызывается:
- При создании/редактировании анкеты.
- Когда пользователя лайкнули или пропустили.
- При создании мэтча — для **обоих** участников.
- Когда приглашённый друг регистрируется (бонус пригласившему).

### `services/cache.py` — Redis-очередь кандидатов

- `get_ranked_candidates` → SQL-запрос кандидатов по L3, с фильтрами пола и возраста.
- Список кладётся в Redis-список `feed:{user_id}`.
- `pop_next` достаёт следующего кандидата. `needs_refill` проверяет, нужна ли перезагрузка.
- Кандидаты, которых уже лайкнули, и скипнутые за последние 24 ч — исключаются из выборки.

## Инфраструктура

- **Docker Compose**: три контейнера — `bot`, `postgres`, `redis`.
- **Alembic**: миграции схемы БД. Миграция `002` добавила поле `username` в `users`.
- **RedisStorage**: FSM-состояния хранятся в Redis — бот остаётся stateless между рестартами.
- **DatabaseMiddleware** (`middlewares/db.py`): инжектирует `AsyncSession` в каждый хэндлер автоматически.
